# UI

## UI Type

Web browser — server-rendered HTML via Jinja2 templates. Minimal vanilla CSS for usability. No JavaScript framework in v0.1.

## Views / Screens

### Screen: Home

**Route:** `GET /`

**Purpose:** Primary entry point showing both first-level entities — Datasets and Sessions — side by side. User can manage datasets and create/open/delete sessions from this one screen.

**Layout:** Two-column layout (stacks on mobile).

**Left column — Datasets:**
- Section title: "Datasets"
- "Add a dataset" inline form:
  - **Dataset name** field (required)
  - Type chooser: "Upload CSV" (default) or "Connect a database"
    - "Upload CSV" → file input
    - "Connect a database" → `dataset_uri` text field; shown **only when external datasets are enabled** (`enable_external_datasets`), otherwise this option is hidden
  - Submit button
- List of Dataset cards, each showing:
  - Name (dataset name), type badge (`parquet` / `postgresql`)
  - "{n} table(s)" count
  - Credential-free URI (rendered via `DatasetURI.display()` — never shows user/password)
  - `connection_error` badge (only when the dataset has a connection error)
  - `last_synced_at` timestamp
  - "Add CSV" button (parquet datasets only) → `POST /datasources/{id}/add-csv`
  - Delete button with inline confirmation ("Delete? Yes / No")
- Empty state if no datasets yet

**Right column — Sessions:**
- Section title: "Sessions"
- "New Session" button → opens inline form below the button showing:
  - Session name field (optional)
  - Checkbox list of all available datasets (must pick ≥ 1; field name stays `data_source_ids`)
  - "Create Session" submit button
  - "Cancel" to close the form
- List of Session cards, each showing:
  - Session name and creation date
  - Dataset chips (names of all attached datasets)
  - Query count
  - "Open" button → `GET /sessions/{id}`
  - Delete button with inline confirmation
- Empty state if no sessions yet

**Actions:**
- Create dataset (upload CSV or connect a database) → `POST /datasources/upload`
- Add CSV to existing parquet dataset → `POST /datasources/{id}/add-csv`
- Delete dataset → `POST /datasources/{id}/delete`
- Create Session → `POST /sessions` (with selected data_source_ids)
- Open Session → `GET /sessions/{id}`
- Delete Session → `POST /sessions/{id}/delete`

### Screen: Session (Chat)

**Route:** `GET /sessions/{session_id}`

**Purpose:** The main interaction screen. Shows all attached datasets, a question form, and all past Q&A for this session inline.

**Key elements:**
- Breadcrumb: Home › Session name
- Session header: session name + chips for each attached dataset (each chip shows the dataset name and its table count)
- "Ask a question" card (always on top):
  - Text input
  - Submit button
  - Loading overlay while pipeline runs
- All past Q&A cards inline (newest first), each showing:
  - Question, answer, SQL iterations, token usage, cost
  - Collapsible agent reasoning trace
  - Highlighted with indigo border if it is the `?new=` record
- Stub-mode banner

**Actions:**
- Submit question → `POST /sessions/{session_id}/query`

---

### Screen: Error

**Route:** rendered inline on 4xx/5xx

**Key elements:** Error message + "Back" link.

---

## Error States

- Pipeline errors: render `error.html` — never bare 500
- File upload errors: flash message on home
- 404: render `error.html` with "not found" message

## Tech Stack

Jinja2 + FastAPI. Minimal inline CSS. No JS frameworks. Stub-mode banner via `llm_provider` context variable.
