# UI

> **Status: deferred to a later phase.** This release ships the HTTP/SSE API only
> ([`05-api.md`](05-api.md)); the chat UI below is the target design for the Next.js phase
> (see [`01-vision.md`](01-vision.md) § Future Phases), not yet built.

## UI Type

A **chat interface** — Next.js + React + Tailwind single-page app talking to the FastAPI backend over
HTTP + SSE ([`05-api.md`](05-api.md)). One dataset workspace on the left, a multi-turn chat on the right.

## Views / Screens

### Screen: Dataset Picker & Creator (left panel)

**Purpose:** Choose an existing dataset to chat about, or create a new one and upload CSVs into it.

**Key elements:**
- List of datasets (name + file count), from `GET /datasets`.
- "New dataset" control: name input → `POST /datasets`.
- File upload control (drag-drop / file chooser) for the selected dataset → `POST
  /datasets/{id}/files`. Accepts CSV only.
- Per-file schema summary (filename, row count, columns + inferred types) from `GET /datasets/{id}`.

**Actions available:**
- Create a dataset.
- Upload one or more CSV files into the selected dataset.
- Select a dataset to start/continue a conversation.

### Screen: Chat Panel (right panel)

**Purpose:** Ask natural-language questions about the selected dataset and read answers — the core
multi-turn loop.

**Key elements:**
- Scrollable message thread (user questions + assistant answers), loaded from `GET
  /conversations/{id}/messages`.
- Each assistant turn renders: the plain-English **answer**, a **result table** (columns + rows), and
  a collapsible **live agent trace** (one line per step's `description` — never raw SQL alone).
- A question input box + send button → `POST /conversations/{id}/messages` (SSE).
- "New conversation" control bound to the selected dataset.

**Actions available:**
- Send a question (first or follow-up — follow-ups build on prior turns).
- Expand/collapse the agent trace for a turn.
- Start a new conversation on the current dataset.

### Live streaming behaviour

While a question is in flight, the chat consumes the SSE stream from `POST
/conversations/{id}/messages`:
- `step` events append to a live trace (a glass box, not a hidden spinner —
  [`../engineering/patterns/react-agent.md`](../engineering/patterns/react-agent.md) § User
  transparency).
- `answer` renders the final text + result table.
- `done` closes the turn; `error` shows the error inline.

## Error States

- **Upload errors** (`UNSUPPORTED_FORMAT`, `CSV_PARSE_ERROR`) — shown inline next to the upload control
  with the envelope's message; the dataset is left unchanged.
- **Dataset not loaded** (`DATASET_NOT_LOADED`, 409 after a restart) — the chat shows a clear,
  actionable banner prompting re-upload/re-materialize, not a generic failure.
- **Run/LLM errors** (`RUN_FAILED`, `LLM_UNAVAILABLE`) — rendered as an inline assistant error bubble
  in the thread.
- **Loading states** — dataset list, upload, and in-flight questions show explicit loading indicators;
  the in-flight question shows the live step trace rather than an opaque spinner.

## Tech Stack

Next.js 15 + React 19 + Tailwind. SSE consumed via the Fetch streaming API. (Exact versions pinned by
the tech-designer in [`../engineering/tech-stack.md`](../engineering/tech-stack.md).)
