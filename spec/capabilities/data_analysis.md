# Capability: Data Analysis

## What It Does

Accepts a plain-English question about an uploaded CSV/Excel file (or a connected PostgreSQL database in Phase 2), generates and locally executes Python/Pandas code or SQL, and returns a text answer plus a Plotly.js interactive chart specification.

## Inputs

| Input | Type | Source | Required |
|-------|------|---------|----------|
| `file_id` | `str` (UUID) | API request body; references `uploaded_files.id` in SQLite | Yes (Phase 1–2 CSV/Excel); No when `source_type=postgres` (Phase 2) |
| `question` | `str` | API request body; plain-English question from the user | Yes |
| `session_id` | `str \| None` | API request body; groups turns for multi-turn context (Phase 3) | No |
| `source_type` | `str` | Derived by `load_dataset` from the file record (`"csv"`, `"excel"`, `"postgres"`) | Derived |

## Outputs

| Output | Type | Destination |
|--------|------|-------------|
| `answer_text` | `str` | API response body; rendered as text in the AnswerCard in the chat UI |
| `chart_spec` | `dict \| None` | API response body; passed directly to `Plotly.newPlot()` in the frontend ChartPanel |
| `run_id` | `str` (UUID) | API response body; references `analysis_runs.id` for history retrieval |
| `error` | `str \| None` | API response body; non-null only on failure; displayed as a red alert card |

## External Calls

| System | Operation | On Failure |
|--------|-----------|------------|
| SQLite (app DB) | Look up `file_path` + `schema_json` from `uploaded_files` by `file_id` | Set `error = "File not found"`; run status `"failed"` |
| Local filesystem | `pandas.read_csv()` or `pandas.read_excel()` | Set `error = "Cannot read file: <exc>"`; run status `"failed"` |
| Local exec() sandbox | `exec(generated_pandas_code, restricted_globals)` with 30 s timeout | Set `error = "Code execution failed: <exc>"`; run status `"failed"` |
| Google Gemini API (`gemini-2.5-flash`) | `plan_analysis` LLM call — generate Pandas/SQL code from schema + question | Retry ×2 with backoff; then set `error`; run status `"failed"` |
| Google Gemini API (`gemini-2.5-pro`) | `reason_answer` LLM call — generate text answer + chart spec from result sample | Retry ×2 with backoff; then set `error`; run status `"failed"` |
| PostgreSQL (Phase 2) | `psycopg2.connect(POSTGRES_DSN)` + `cursor.execute(generated_sql)` | Set `error = "PostgreSQL query failed: <exc>"`; run status `"failed"` |

## Business Rules

- **No raw data to LLM.** Only a sample of at most 500 rows (serialized as CSV) from the execution result is included in the `reason_answer` LLM prompt. Full dataset rows are never sent to any external API.
- **Pandas code constraints.** The code generated by `plan_analysis` must: (a) use variable `df` (injected into exec scope), (b) assign the final result to a variable named `result`, (c) produce a result that is either a `pandas.DataFrame` or a scalar (int/float/str). It must not: import `os`, `sys`, `subprocess`; call `open()`, `socket`, `requests`, `urllib`; write to files or the network.
- **Static code validation.** Before execution, `execute_code` performs an AST check on the generated code using Python's `ast` module. Any forbidden import statement or call causes the run to fail immediately with a clear error message — the code is never executed.
- **Execution timeout.** The `exec()` call is wrapped in a 30-second timeout using `concurrent.futures.ThreadPoolExecutor`. A timeout is treated as a code execution failure.
- **Chart type selection.** The `reason_answer` node (via its system prompt) selects chart type as follows:
  - `bar` — categorical comparisons, top-N rankings, counts by group
  - `line` — time-series or trends over an ordered sequence
  - `scatter` — correlations between two continuous numeric variables
  - `pie` — part-of-whole proportions with ≤6 categories; `bar` is used instead for >6 categories
  - `null` — scalar answer or free-text where no chart adds value
- **Schema introspection.** `load_dataset` extracts `schema_info` containing: list of column names, dict of `{column_name: dtype_string}`, and the first 3 rows as a list of lists. This is what gets sent to the LLM — never the full dataset.
- **Result trimming.** If the execution result is a DataFrame with more than 500 rows, it is trimmed to 500 rows via `.head(500)` before serialization. This trimmed CSV is the only data included in the LLM prompt.
- **Run persistence.** Every analysis run (success or failure) is recorded in `analysis_runs` with `status`, `answer_text`, `chart_spec_json`, and `error_message`. The run record is created by `finalize` (success) or `handle_error` (failure).
- **File ID scoping.** The `file_id` in the request must reference a row in `uploaded_files`. If the file has been deleted from the filesystem or the SQLite record is missing, the run fails with a clear "File not found" error.

## Success Criteria

- [ ] Uploading a CSV file via `POST /api/files/upload` returns a `file_id` and a `schema_preview` containing column names, dtypes, and 3 sample rows within 5 seconds.
- [ ] `POST /api/analysis/run` with a valid `file_id` and a plain-English question returns `{run_id, answer_text, chart_spec}` where `answer_text` is non-empty and `chart_spec` conforms to the Plotly spec defined in `spec/architecture.md` → Chart Pipeline.
- [ ] The `chart_spec.data` array contains at least one trace object with a `type` field matching one of `bar`, `line`, `scatter`, `pie`.
- [ ] The generated Pandas code passes the AST check (no forbidden imports) and executes without error on a standard CSV with mixed column types (strings + integers + dates).
- [ ] Only `result_sample` (≤500 rows as CSV) appears in the LLM prompt; the full DataFrame is never serialized into a prompt (verifiable by inspecting the LangSmith trace).
- [ ] A question that would require more than 500 rows of result data is correctly trimmed and the answer still reflects the aggregated data, not just the first 500 rows.
- [ ] On a Gemini API error, the run returns `{"error": "<message>", "status": "failed"}` and the `analysis_runs` row has `status="failed"` and a non-null `error_message`.
- [ ] On generated code that attempts a forbidden import (e.g. `import os`), the run fails with `error = "Code validation failed: forbidden import 'os'"` — the code is never executed.
- [ ] `GET /api/analysis/runs` returns a list including the completed run with `answer_text`, `chart_spec_json` (non-null), and `status="completed"`.
- [ ] The Playwright E2E test confirms: upload a CSV → type a question → an AnswerCard appears with non-empty text and a rendered Plotly chart (chart `<div>` is present and non-empty).
