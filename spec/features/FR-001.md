# FR-001 — Data Analyst Agent

**Date:** 2026-06-21  
**Author:** researcher  
**Status:** approved

---

## Problem Statement

Analysts and technical users frequently need to interrogate heterogeneous datasets — CSVs from exports, Excel files from partners, JSON from APIs, Parquet from data pipelines — without spinning up a full BI stack or writing ad-hoc SQL manually. The round-trip of "load file → write query → format result → build chart → interpret" is slow, error-prone, and requires context-switching across tools. This feature delivers a conversational agent that owns that entire loop: it accepts natural-language questions, plans and executes DuckDB queries, interprets the results, and returns formatted text, tables, or Plotly charts — simulating the workflow of a senior data analyst. Every operation is audited. Sessions persist across restarts so work is never lost.

## Target Users

| User | Context | Primary need |
|------|---------|--------------|
| Data analyst (technical) | Has datasets in CSV/Excel/JSON/Parquet; wants quick answers without writing SQL | Ask NL questions, get charts and tables immediately |
| Developer / product manager (semi-technical) | Needs to explore a dataset without knowing SQL or Python | Upload a file and ask plain-English questions |
| Researcher | Running repeated analyses across multiple loaded datasets in a single session | Persistent session so context and datasets survive restarts |

## Success Criteria

- [ ] User loads a CSV, Excel, JSON, or Parquet file via POST /datasets — it is registered as a named dataset and queryable within the same request cycle.
- [ ] NL query "show top 10 rows of [dataset]" returns a formatted Markdown table rendered in the web UI.
- [ ] NL query "plot [metric] over [dimension]" returns a Plotly JSON spec rendered as an interactive chart in the web UI.
- [ ] Session (datasets registered + conversation history) persists across agent restarts — re-starting the server does not lose registered datasets or prior conversation turns.
- [ ] Every SQL execution is logged to the audit_log table: timestamp, session_id, query_text, rows_affected, duration_ms.
- [ ] All unit tests pass with GEMINI_API_KEY=stub (offline mode — no live LLM calls required for the test suite).

## Non-Goals

- Multi-user authentication — single-user local session only for this milestone.
- Real-time or streaming data sources — static file uploads only.
- Scheduling or alerting on data changes.
- Export to Excel or PDF.
- Image-based chart export (server-side rendering of charts to PNG/SVG).

## Key Constraints

| Constraint | Value | Reason |
|-----------|-------|--------|
| Stack | Python 3.12 + uv, FastAPI (port 8001), LangGraph, DuckDB, Next.js (port 3000) | Approved by user; matches harness default Python stack |
| LLM | Gemini 2.5 Flash via google-generativeai SDK | Token-economical; user-specified model |
| Database | DuckDB — stores datasets, session state, audit log | Native Parquet/CSV/JSON/Excel ingestion; no separate DB process |
| Data sources | CSV, Excel (.xlsx), JSON, Parquet | DuckDB reads all four natively |
| Frontend | Next.js thin shell — renders Plotly charts, Markdown tables, plain text | User-specified output medium |
| API keys needed | GEMINI_API_KEY | Required at runtime; stub value sufficient for offline tests |
| Deploy target | Local demo first; Render on request | User-specified |
| Ports | Backend 8001, Frontend 3000 | Harness default; no conflict expected |
