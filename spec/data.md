# Data Model

## Storage Technology

**SQLite** (via SQLAlchemy 2.0, Alembic migrations) at `data/agent.db`. SQLite is the app metadata store only — it holds the file registry, schema cache, and run history. It is never used as a substitute for PostgreSQL; PostgreSQL is an external data source queried at analysis time (Phase 2) and has no SQLAlchemy models in this app.

Raw uploaded files (CSV/Excel) are stored on the local filesystem at `data/uploads/<uuid>/<original_filename>` and referenced by path in `uploaded_files.file_path`.

---

## Entities

### Entity: `runs` (existing skeleton — retained)

The skeleton's existing run table, retained for backward compatibility and the existing `/runs` endpoint.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | TEXT (UUID) | yes | Primary key; generated with `uuid4()` |
| `status` | TEXT | yes | `"pending"` \| `"completed"` \| `"failed"` |
| `input_text` | TEXT | no | Original input (skeleton use only) |
| `output_text` | TEXT | no | Output text (skeleton use only) |
| `error_message` | TEXT | no | Error string if status is `"failed"` |
| `created_at` | TIMESTAMP WITH TIME ZONE | yes | UTC creation time |
| `updated_at` | TIMESTAMP WITH TIME ZONE | yes | UTC last-update time |

---

### Entity: `uploaded_files`

Represents a CSV or Excel file uploaded by the user. One row per uploaded file. The parsed schema is cached here so the file does not need re-parsing on every question.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | TEXT (UUID) | yes | Primary key; generated with `uuid4()` |
| `filename` | TEXT | yes | Stored filename on disk (may include UUID prefix to avoid collisions) |
| `original_name` | TEXT | yes | Original filename as uploaded by the user (for display) |
| `file_path` | TEXT | yes | Absolute path to the file on the local filesystem |
| `source_type` | TEXT | yes | `"csv"` \| `"excel"` — determines how `load_dataset` reads the file |
| `schema_json` | TEXT (JSON) | yes | Serialized schema: `{"columns": [...], "dtypes": {...}, "sample_rows": [[...]]}` — first 3 rows |
| `row_count` | INTEGER | no | Approximate row count of the file (populated on upload, best-effort) |
| `file_size_bytes` | INTEGER | no | File size in bytes |
| `created_at` | TIMESTAMP WITH TIME ZONE | yes | UTC upload time |

---

### Entity: `analysis_runs`

One row per user question — the full history of analysis requests. Created by the `finalize` or `handle_error` node at the end of each graph invocation.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | TEXT (UUID) | yes | Primary key; generated with `uuid4()` |
| `file_id` | TEXT (UUID) | no | Foreign key → `uploaded_files.id`; NULL when source is PostgreSQL (Phase 2) |
| `session_id` | TEXT | no | Optional session grouping string for multi-turn context (Phase 3) |
| `question` | TEXT | yes | The user's plain-English question |
| `answer_text` | TEXT | no | The LLM-generated text answer; NULL on failure |
| `chart_spec_json` | TEXT (JSON) | no | Serialized Plotly chart spec (see Chart Pipeline in `spec/architecture.md`); NULL if no chart or on failure |
| `generated_code` | TEXT | no | The Pandas/SQL code generated by `plan_analysis`; stored for debugging |
| `result_sample_csv` | TEXT | no | The ≤500-row CSV sample sent to the LLM; stored for auditability |
| `conversation_history_json` | TEXT (JSON) | no | Phase 3: serialized prior turns `[{question, answer_text, chart_spec}, ...]` |
| `status` | TEXT | yes | `"pending"` \| `"running"` \| `"completed"` \| `"failed"` |
| `error_message` | TEXT | no | Error string if status is `"failed"`; NULL otherwise |
| `created_at` | TIMESTAMP WITH TIME ZONE | yes | UTC time the run was initiated |
| `completed_at` | TIMESTAMP WITH TIME ZONE | no | UTC time `finalize` or `handle_error` completed |

---

### Entity: `pg_connections` (Phase 2)

Named PostgreSQL connection configurations. One row per saved connection. The DSN is **not** stored here — it is read at connection time from `POSTGRES_DSN` in `.env`. This table stores only the connection label and the introspected schema for display in the UI.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | TEXT (UUID) | yes | Primary key; generated with `uuid4()` |
| `name` | TEXT | yes | User-assigned display name (e.g. "My Analytics DB") |
| `schema_json` | TEXT (JSON) | no | Cached table and column schema from the last successful connection; `{"tables": [{"name": "...", "columns": [...]}]}` |
| `last_connected_at` | TIMESTAMP WITH TIME ZONE | no | UTC time of the last successful connection test |
| `created_at` | TIMESTAMP WITH TIME ZONE | yes | UTC time this connection was registered |

> **Assumed:** Only one PostgreSQL connection is supported in Phase 2 (single personal user). `pg_connections` may have at most one active row; adding a new one replaces the previous. The DSN is always sourced from `POSTGRES_DSN` in `.env` — it is never stored in this table.

## Relationships

```
uploaded_files (1) ──────────────── (0..N) analysis_runs
                                         via analysis_runs.file_id → uploaded_files.id

pg_connections (0..1) ─── (no FK) ─── analysis_runs
                                         (when file_id IS NULL, source is PostgreSQL)
```

- `analysis_runs.file_id` is a nullable foreign key to `uploaded_files.id`. When the source is PostgreSQL (Phase 2), `file_id` is NULL and the connection is implicit via `POSTGRES_DSN`.
- `pg_connections` has no foreign key into `analysis_runs` — the connection is identified by the presence of `POSTGRES_DSN` in `.env` at query time.

## Data Lifecycle

| Entity | Created | Updated | Deleted |
|--------|---------|---------|---------|
| `uploaded_files` | On `POST /api/files/upload` | Never (immutable after upload) | Not deleted automatically; manual cleanup only |
| `analysis_runs` | At start of `run_agent()` (pending); finalized by `finalize` or `handle_error` | `status`, `answer_text`, `chart_spec_json`, `error_message`, `completed_at` updated at graph end | Never deleted automatically |
| `pg_connections` | On `POST /api/pg/connect` (Phase 2) | `schema_json`, `last_connected_at` updated on reconnect | Replaced when a new connection is registered |
| `runs` (skeleton) | On `POST /runs` (skeleton endpoint) | `status`, `output_text`, `error_message` updated at run end | Never |

Files on the local filesystem (`data/uploads/`) are retained indefinitely. No automatic cleanup or TTL is implemented.

## Sensitive Data

| Field | Sensitivity | Protection |
|-------|------------|------------|
| `uploaded_files.file_path` | Low — local path only; no cloud exposure | Stored in SQLite; accessible only locally |
| `POSTGRES_DSN` (env var) | High — contains DB credentials | Stored only in `.env` (gitignored); never in SQLite, never logged |
| `analysis_runs.result_sample_csv` | Medium — contains a sample of user data | Stored in SQLite locally; not sent to any cloud service except the ≤500-row sample to Gemini API |
| `analysis_runs.generated_code` | Low | Stored in SQLite for debugging; not exposed via API by default |

No PII is collected or stored beyond what the user uploads in their own data files.
